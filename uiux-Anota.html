Project Overview

Project outline / Clients’ names / Project goals / Major deliverables

Objective:

Improve medical data quality and reduce medical* annotation work time

Overview:

Use medical data to help health professionals improve the quality of care and patient experience, make better decisions, and improve healthcare outcomes.

Medical annotations capture a structured representation of knowledge stored in unstructured text. The ultimate goal of the task of mapping text to structured knowledge is to feed the annotations into a machine learning algorithm that learns how to automatically extract the medical knowledge contained in the text.

NHIA (National Health Insurance Administration of Taiwan) is dedicated to create high quality medical annotations. Traditionally, annotation project managers (primary users) interpret the annotated data and annotators (secondary users) are asked to document the process and infrastructure of medical annotation. However, many project managers do not provide a comprehensive description of the project. Annotation of data is often inefficient and incorrect. Therefore, team members need more time to go over all the data that the *NLP model cannot understand and use.

In the first stage of the annotation software design, the team focused on the survey of the needs of the medical annotation projects, including medical content, processes, infrastructure, and logic input. At a later stage, the team focuses on how to use the UI interface and the process of passing usability testing. Most project managers are not familiar with computers, so the interface is designed to be simple and intuitive. Users can easily accomplish their tasks with minimal effort.

Key users of Anota say that the software provides tremendous support in improving data quality and reducing annotation work time. Anota will be released online in the future.

Starting the Design

Includes problems, solutions, flowcharts, wireframes to get you started

Problem:

Help Annotation Project Managers (primary users) develop medical annotation projects and assign tasks to annotators through a simplified graphical user interface. This way, annotators can see the big picture of the annotated project and annotate text accurately and correctly to improve overall productivity.

Traditionally, most project managers have only provided annotation files to the annotators without explaining the project framework, medical knowledge, annotation terminology, procedures, and details. Furthermore, annotation tasks are assigned verbally, which often leaves the annotator unaware of how to complete the task.

As such, Anota focuses on mining insights from stakeholders on plans and feedback (questions, concerns, or ideas) for annotation projects. Anota helps annotation teams start projects faster and more efficiently.

Solutions:

Allows the project managers to provide clear and comprehensive information about the projects with all required documentation attached. Project managers simply follow the steps on the UI. They will have a clear infrastructure outlining the timeline and steps of the project. 

Step-By-Step Instructions:

To give users clear annotation instructions, Anota provides a comprehensive description of how to create an annotation project. With the help of the logical planning provided by Anota, the project manager just needs to follow the step-by-step instructions to complete the task.

Simplified Graphical User Interface:

Anota's interface is simplified without unnecessary frills and visuals. Users only need to interact with the interface and follow the given instructions when annotating items.

Flowchart:

The project manager mentioned that they are not familiar with creating projects with software, so they need a logical planning process based on usage scenarios and usage behaviours, but the steps should be limited.

Wireframe:

At the beginning of creating the wireframe, the idea of ​​putting all the flow on one page was also brought up. However, users said they were not used to too many instructions on the screen. Users are confused and conflicted and need more time to complete projects.

So, to solve this problem, the users can only do one thing on a page. After completing a step, they will go to the next step (page). However, users reported that they were sometimes very nervous and wondered how many steps and pages were left to complete the project. Therefore, a progress bar is added to inform the user how many steps need to be completed.

Refining the design

Utilize visual design to help users access and understand user interfaces

UI Design:

This UI is designed to facilitate the seamless completion of the task at hand and to be aesthetically pleasing to the user. Therefore, keeping the UI clean and simple is a priority. Additionally, to let users know what the process requires, a progress step bar is designed to let users know where they are in the process.

Colour:

Blue is usually associated with calmness and focus. Therefore, the blue colour was chosen to allow the user to focus on building the annotation project.

Typography:

Our design team wanted users to read the text effortlessly and make the most of the message we wanted to convey. Therefore, the font needs to be legible. Noto Sans was applied in Chinese, whiled Calibri was in English because of their readability and legibility on web and mobile user interfaces.

Going Forward

User Feedback:

Users are full of praise for Anota, saying that Anota not only helps them manage their annotation projects efficiently but also improves annotation quality by about 2%. Due to covid-19, users have to work remotely. So they want the app to be online so that it can be used anywhere.

Limitation & Reflection:

The contract with NHIA states that the team was only allowed to interview and test with a small number of project managers. However, most insights should be provided by the annotators, not their supervisors (project managers). That is, when the annotator is confronted with the precise problem, we cannot get the full picture of the complete scene. As a result, challenges, questions, feedback, and assumptions from real users (project managers) may still be wrong.
